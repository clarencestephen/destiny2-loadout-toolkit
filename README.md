# Destiny 2 Loadout Toolkit

A personal Destiny 2 planner ‚Äî DIM share URL decoder, wishlist tracker, exotic mission queue, and build/mod recommender ‚Äî all in one Excel workbook driven by a Python install wizard.

---

## What you get

After running the install wizard you'll have:

- **`my_loadouts.xlsx`** ‚Äî a workbook with 9 sheets:
  - `START HERE` ‚Äî onboarding instructions
  - `PRIORITIES` ‚Äî what to chase next (exotics, campaigns, farming routes, mechanics, mods)
  - `EXOTIC MISSIONS` ‚Äî current rotation + legacy farms, seeded with 10 missions
  - `WISHLIST` ‚Äî DIM-CSV-compatible columns
  - `HUNTER BUILDS` / `TITAN BUILDS` / `WARLOCK BUILDS` ‚Äî one row per build, seeded with 3 examples each
  - `MOD REFERENCE` ‚Äî current artifact perks + **auto-populated mod recommendations** based on your archetype + goals
  - `DIM LOADOUTS (FULL)` ‚Äî auto-generated by `decode_dim.py` from your DIM share URLs

- **`user_config.json`** ‚Äî your personal config (API key, DIM URLs, build focus). **Gitignored ‚Äî never committed.**

## Install ‚Äî two ways

### 1. GUI installer  (recommended, Windows-friendly)

```bash
git clone https://github.com/clarencestephen/destiny2-loadout-toolkit
cd destiny2-loadout-toolkit
pip install -r requirements.txt
python setup_gui.py
```

A windowed wizard walks you through:
1. **Welcome** ‚Äî what you'll need
2. **Bungie API key** ‚Äî opens the portal in your browser, you paste the key back
3. **Your Bungie name + primary class**
4. **Build focus** ‚Äî pick an armor archetype + 1+ build goals
5. **DIM loadouts** ‚Äî add as many DIM share URLs as you want (or skip and add later)
6. **Confirm + install** ‚Äî writes config, builds workbook, runs the decoder

### 2. CLI installer  (Mac/Linux/headless, or if tkinter isn't available)

```bash
python3 setup.py
```

Same flow, terminal-based. Both write the same `user_config.json` and `my_loadouts.xlsx`.

### 3. Pre-built Windows .exe  (no Python required)

If you don't want to install Python, grab the latest `.exe` from the [Releases page](https://github.com/clarencestephen/destiny2-loadout-toolkit/releases). Double-click and follow the wizard. The .exe is built by GitHub Actions on every tagged release ‚Äî see `.github/workflows/build-windows-exe.yml`.

## What you need before installing

1. **A Bungie API key** ‚Äî free, 30 seconds. See [docs/getting-bungie-api-key.md](docs/getting-bungie-api-key.md). The wizard will open the portal for you in step 2.
2. **Your DIM share URLs** *(optional at install)* ‚Äî one per loadout you want decoded. See [docs/getting-dim-share-urls.md](docs/getting-dim-share-urls.md). You can add these during install or later with `python add_loadout.py`.
3. **Python 3.9+ and pip** (if not using the pre-built .exe).

## Day-to-day commands

| Command | What it does |
|---|---|
| `python decode_dim.py` | Re-fetch every DIM URL in your config and rewrite the `DIM LOADOUTS (FULL)` sheet. Run this whenever you change a loadout in DIM. |
| `python add_loadout.py` | Append a new DIM share URL to your config without editing JSON. Then auto-runs the decoder. |
| `python init_workbook.py` | Rebuild the workbook template from scratch. **Asks before overwriting** ‚Äî but if you've manually edited sheets, back up first. |
| `python setup.py` | Re-run the install wizard. Safe ‚Äî asks before overwriting anything. |
| `python demo_examples.py` | Try the toolkit with 27 community-shared loadouts before configuring your own. Builds an `example_workbook.xlsx` next to your real one. See [examples/README.md](examples/README.md). |

## How your data is protected

- **Bungie API key** is stored in `user_config.json`, which is `.gitignore`d. It never leaves your machine and is never committed.
- **Your DIM URLs** also live in `user_config.json`. Sharing them is harmless (they're already public share links), but they don't end up in this repo.
- **Your workbook** (`my_loadouts.xlsx`) is also gitignored. Bring your own.

The shipped repo contains only the toolkit code, an empty workbook template, and example placeholders ‚Äî no personal data.

## DIM exports ‚Äî share URLs vs CSVs

This toolkit uses two different DIM features:

- **DIM Share URLs** (`https://dim.gg/...`) ‚Üí decoded into the `DIM LOADOUTS (FULL)` sheet. One URL per loadout.
- **DIM Inventory CSVs** (`destinyWeapons.csv` etc.) ‚Üí paste rows into the `WISHLIST` sheet to track what you own / want.

How to get the CSVs:

![DIM Spreadsheets settings page](docs/images/dim-spreadsheets-page.png)

In DIM: **Settings ‚Üí Spreadsheets ‚Üí click `Weapons`, `Armor`, or `Ghost`** to download a CSV. Detailed steps in [docs/getting-dim-csv-export.md](docs/getting-dim-csv-export.md).

## Customizing for yourself

Everything in the workbook is meant to be edited:

- **PRIORITIES** ‚Äî replace the seed rows with your own exotic chase list.
- **HUNTER / TITAN / WARLOCK BUILDS** ‚Äî each sheet ships with 3 example rows in italic grey. Type over them.
- **EXOTIC MISSIONS** ‚Äî set `Status` to `DONE` / `TODO` / `In Progress` and re-sort by `Priority`.
- **WISHLIST** ‚Äî paste rows from your DIM CSV export.
- **MOD REFERENCE** ‚Äî auto-seeded from your setup answers, but you can edit it freely. Re-run `python init_workbook.py` (with confirmation) to regenerate from scratch.
- **DIM LOADOUTS (FULL)** ‚Äî **don't edit by hand.** It's overwritten on every `decode_dim.py` run. Edit the source DIM loadout in DIM instead, then re-run the decoder.

See [docs/customizing-the-workbook.md](docs/customizing-the-workbook.md) for a sheet-by-sheet rundown.
