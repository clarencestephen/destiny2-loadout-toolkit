# The Way of the Sith

> **Destiny voyager optimizer · wishlist organizer · API stat tracking**

A personal Destiny 2 planner — DIM share URL decoder, wishlist tracker, exotic mission queue, and build/mod recommender — all in one Excel workbook driven by a Python install wizard.

---

## What you get

After running the install wizard you'll have:

- **`my_loadouts.xlsx`** — a workbook with 9 sheets:
  - `START HERE` — onboarding instructions
  - `PRIORITIES` — what to chase next (exotics, campaigns, farming routes, mechanics, mods)
  - `EXOTIC MISSIONS` — current rotation + legacy farms, seeded with 10 missions
  - `WISHLIST` — DIM-CSV-compatible columns
  - `HUNTER BUILDS` / `TITAN BUILDS` / `WARLOCK BUILDS` — one row per build, seeded with 3 examples each
  - `MOD REFERENCE` — current artifact perks + **auto-populated mod recommendations** based on your archetype + goals
  - `DIM LOADOUTS (FULL)` — auto-generated by `decode_dim.py` from your DIM share URLs

- **`user_config.json`** — your personal config (API key, DIM URLs, build focus). **Gitignored — never committed.**

## Install — three ways

### 1. GUI installer  (recommended, Windows-friendly)

```bash
git clone https://github.com/clarencestephen/the-way-of-the-sith
cd the-way-of-the-sith
pip install -r requirements.txt
python setup_gui.py
```

A windowed wizard walks you through:
1. **Welcome** — what you'll need
2. **Bungie API key** — opens the portal in your browser, you paste the key back
3. **Your Bungie name + primary class**
4. **Build focus** — pick an armor archetype + 1+ build goals
5. **DIM loadouts** — add as many DIM share URLs as you want (or skip and add later)
6. **Confirm + install** — writes config, builds workbook, runs the decoder

### 2. CLI installer  (Mac/Linux/headless, or if tkinter isn't available)

```bash
python3 setup.py
```

Same flow, terminal-based. Both write the same `user_config.json` and `my_loadouts.xlsx`.

### 3. Pre-built Windows .exe  (no Python required)

If you don't want to install Python, grab the latest `.exe` from the [Releases page](https://github.com/clarencestephen/the-way-of-the-sith/releases). Double-click and follow the wizard. The .exe is built by GitHub Actions on every tagged release — see `.github/workflows/build-windows-exe.yml`.

## What you need before installing

1. **A Bungie API key** — free, 30 seconds. See [docs/getting-bungie-api-key.md](docs/getting-bungie-api-key.md). The wizard will open the portal for you in step 2.
2. **Your DIM share URLs** *(optional at install)* — one per loadout you want decoded. See [docs/getting-dim-share-urls.md](docs/getting-dim-share-urls.md). You can add these during install or later with `python add_loadout.py`.
3. **Python 3.9+ and pip** (if not using the pre-built .exe).

## Day-to-day commands

| Command | What it does |
|---|---|
| `python decode_dim.py` | Re-fetch every DIM URL in your config and rewrite the `DIM LOADOUTS (FULL)` sheet. Run this whenever you change a loadout in DIM. |
| `python add_loadout.py` | Append a new DIM share URL to your config without editing JSON. Then auto-runs the decoder. |
| `python init_workbook.py` | Rebuild the workbook template from scratch. **Asks before overwriting** — but if you've manually edited sheets, back up first. |
| `python setup.py` | Re-run the install wizard. Safe — asks before overwriting anything. |
| `python demo_examples.py` | Try the toolkit with 27 community-shared loadouts before configuring your own. Builds an `example_workbook.xlsx` next to your real one. See [examples/README.md](examples/README.md). |

## How your data is protected

- **Bungie API key** is stored in `user_config.json`, which is `.gitignore`d. It never leaves your machine and is never committed.
- **Your DIM URLs** also live in `user_config.json`. Sharing them is harmless (they're already public share links), but they don't end up in this repo.
- **Your workbook** (`my_loadouts.xlsx`) is also gitignored. Bring your own.

The shipped repo contains only the toolkit code, an empty workbook template, and example placeholders — no personal data.

## DIM exports — share URLs vs CSVs

This toolkit uses two different DIM features:

- **DIM Share URLs** (`https://dim.gg/...`) → decoded into the `DIM LOADOUTS (FULL)` sheet. One URL per loadout.
- **DIM Inventory CSVs** (`destinyWeapons.csv` etc.) → paste rows into the `WISHLIST` sheet to track what you own / want.

How to get the CSVs:

![DIM Spreadsheets settings page](docs/images/dim-spreadsheets-page.png)

In DIM: **Settings → Spreadsheets → click `Weapons`, `Armor`, or `Ghost`** to download a CSV. Detailed steps in [docs/getting-dim-csv-export.md](docs/getting-dim-csv-export.md).

## Customizing for yourself

Everything in the workbook is meant to be edited:

- **PRIORITIES** — replace the seed rows with your own exotic chase list.
- **HUNTER / TITAN / WARLOCK BUILDS** — each sheet ships with 3 example rows in italic grey. Type over them.
- **EXOTIC MISSIONS** — set `Status` to `DONE` / `TODO` / `In Progress` and re-sort by `Priority`.
- **WISHLIST** — paste rows from your DIM CSV export.
- **MOD REFERENCE** — auto-seeded from your setup answers, but you can edit it freely. Re-run `python init_workbook.py` (with confirmation) to regenerate from scratch.
- **DIM LOADOUTS (FULL)** — **don't edit by hand.** It's overwritten on every `decode_dim.py` run. Edit the source DIM loadout in DIM instead, then re-run the decoder.

See [docs/customizing-the-workbook.md](docs/customizing-the-workbook.md) for a sheet-by-sheet rundown.
